##  路由的重定向
> 此重定向是更换锚点,并不想后端重定向向服务器请求
+   重定向设置,默认访问路径
例: 当访问到根路径时,默认显示login模块
```
    {path:'/',redirect:'/login'}
```

## 侦听路由,响应路由参数变化
> 在使用路由参数时, 参数的改变, 原来的组件实例会被复用, 也就是说 /login/张三 到 /login/李四, 它们使用的都是匹配 /login 的组件, 此时组件会被复用, 组件被复用会比较高效, 但是这意味着不会调用生命周期函数; 

> 复用组件时, 想对路由的变化做出响应, 可以通过侦听器进行;
```
    const User = {
        template:'...',
        watch:{
            '$route'(to,from){
                // 对路由的变化做出响应
            }
        }
    }
```
