+   自调函数避免命名冲突
    ```
        - 格式: ;(function(window,document,undefined){ ... })(window,document,undefined);

        1. 代码最前面的分号，可以防止多个文件压缩合并因为某个语句最后一行语句没加分号，而引起合并后的语法错误。

        2. window,document对象都是全局环境下的,而在函数体内的window和document实际是局部变量;传递这两个参数的好处是可以提高性能,减少作用域链的查询时间,如果函数体的内部需要多次用到window和document的时候,就可以把这两个参数传进去;如果插件用不到就没有必要了;

        3. undefined在老版本是不被支持的,直接使用会被报错,JS框架考虑到兼容性,因此就添加一个undefined,实参处可以不传递undefined,因为不传递实参在函数内部使用到形参的时候,本身就是undefined;        

        4. 用()强调函数体,避免外部变量发生冲突
    ```

+   代码压缩
    ```
        1. 在浏览器解析js文件的时候,会把当前的js下载下来,每次下次一个js,都会去服务器请求一个服务,请求之后再下载回来; js文件多了以后请求的次数会影响加载的速度;

        2. 但是自调用函数内的东西外部都不可以使用,因此如果要让外部使用,就要定义成window的属性,因为全局的属性都是作为window的属性存在;

        3. 压缩代码: 目的是让代码的体积变小,访问网页的时候速度变快;

        - 注意: 开发的时候是分文件开发的,上线的时候要把代码合并起来;

        4. 把所有的js文件手工合并到一个js文件中;(以后可以使用工具合并);
            例:
                // -------xx模块--------
                ;(function(){})();

                // -------xx模块--------
                ;(function(){})();
    ```

            console.log(1);

        undefined(function(){
            console.log(2);
        })()

        // 系统按顺序执行会把undefined(function(){})看成是调用函数;因为就会报错;

        // 2. 第二种情况

        var fn = fu nction(){}
        (function(){
            console.log(2);
        })()

        // 浏览器解析后,会变成如下形式:
        var fn = function(){}(function(){})()
        // 就等于把匿名函数传递给了前面的function(){}
